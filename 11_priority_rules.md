# Priority Rules

## 목적

여러 작업이 동시에 있을 때 AI가 어떤 순서로 처리하는지 기준을 정의한다.

---

## 기본 우선순위 규칙

1. risk_level: critical → high → medium → low
2. 같은 risk_level이면 Jira 우선순위 (P1 > P2 > P3) 따름
3. 같은 우선순위면 생성일 순서 (먼저 만든 것 먼저)

---

## 의존성 처리

- 작업 간 의존성이 있으면 선행 작업 먼저 처리
- AI가 작업 분해 시 의존성 그래프를 자동 생성
- 독립적인 작업은 병렬 처리 가능

---

## 사람 오버라이드

- 사람이 Jira에서 순서를 변경하면 AI는 즉시 반영
- 긴급 작업이 들어오면 현재 작업 중단 후 전환 가능
