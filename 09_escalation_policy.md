# Escalation Policy

## 목적

AI가 스스로 해결하지 못하는 상황을 감지하고 사람에게 넘기는 기준과 절차를 정의한다.

---

## 자동 에스컬레이션 조건

- AI가 동일 작업을 3회 이상 재시도해도 테스트 실패
- AI가 요구사항을 해석할 수 없다고 판단 (모호성 감지)
- 가드레일 영역에 변경이 필요한데 초안에 명시되지 않은 경우
- CI/CD 파이프라인이 2회 연속 실패

---

## 에스컬레이션 방식

- Jira 티켓에 `needs-human` 라벨 자동 부착
- 담당자에게 알림 (Slack / 이메일)
- AI가 시도한 내용과 실패 원인을 요약해서 첨부

---

## 에스컬레이션 후 흐름

### 방식 1: 사람이 방향 제시

사람이 해결 방향을 코멘트로 작성하면 AI가 이어서 작업한다.

```
AI 실패 → 에스컬레이션 → 사람 코멘트 → AI 재작업
```

### 방식 2: 사람이 직접 구현

사람이 직접 코드를 수정한 후 AI에게 리뷰를 요청한다.

```
AI 실패 → 에스컬레이션 → 사람 직접 구현 → AI 리뷰
```
