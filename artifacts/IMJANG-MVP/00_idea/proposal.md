## 제목

임장 기록 앱 — 실거주 중심 부동산 임장 관리 플랫폼

---

## 문제/배경

부동산 매수 희망자가 아파트 단지를 직접 방문(임장)하여 수집한 정보를 **체계적으로 기록하고 비교할 수단이 없다.** 현재는 메모장, 사진첩, 엑셀 등에 분산 저장되어 다음 문제가 발생한다:

1. **정보 파편화** — 단지별 체크항목, 사진, 메모가 별도 앱에 흩어져 검색/비교 불가
2. **비교 어려움** — 여러 단지를 동일 기준으로 나란히 비교할 도구 부재
3. **공유 불편** — 가족/지인과 임장 결과를 공유하려면 수동 정리 필요
4. **지도 기반 탐색 부재** — 방문 단지와 관심 단지를 지도 위에서 한눈에 볼 수 없음
5. **공공데이터 단절** — 실거래가, 단지 기본정보 등 공공데이터를 직접 검색해야 함

| 임장 기록 | 체크항목 평점(소음/경사/상권/주차/일조권), 장단점/총평, 사진 첨부(최대 20장), 오프라인 기록 |

## 상세 요구사항 (Requirements)

### 인증 (Auth)

- **FR-AUTH-01~02:** 이메일/비밀번호 회원가입 및 로그인 (유효성 검사, 에러 메시지, 세션 유지)
- **FR-AUTH-03~04:** Google/Apple 소셜 로그인 (Apple은 iOS만)
- **FR-AUTH-05:** 로그아웃 (확인 다이얼로그)
- **FR-AUTH-06:** 닉네임(2~20자) + 프로필 사진 관리

### 단지 관리 (Complex)

- **FR-COMP-01:** 공공API 기반 단지 검색/자동완성 (2글자 이상, 500ms 디바운스) + 등록
- **FR-COMP-02:** 단지 기본 정보 표시 (세대수, 동수, 층수, 난방, 사용승인일, 건설사, 용적률/건폐율)
- **FR-COMP-03:** 단지 상태 관리 (관심/방문예정/방문완료/재방문/제외)
- **FR-COMP-04:** 단지 목록 조회 (상태 필터, 정렬, 무한 스크롤)
- **FR-COMP-05:** 단지 삭제 (연관 데이터 연쇄 삭제, Owner만 가능)
- **FR-COMP-06:** 실거래가 자동 조회 (최근 6개월, 캐싱 + 폴백)

### 임장 기록 (Inspection)

- **FR-INSP-01:** 임장 기록 작성 (방문일/시간대/체크항목 평점 1~5/장단점/총평/종합 평점)
- **FR-INSP-02:** 사진 첨부 (최대 20장, 자동 압축 1920px/80%, 캡션, 업로드 진행률)
- **FR-INSP-03~04:** 임장 기록 조회/수정
- **FR-INSP-05:** 임장 기록 삭제 (Storage 사진 연쇄 삭제)
- **FR-INSP-06:** 오프라인 임장 기록 (텍스트: Firestore 오프라인 자동 동기화 / 사진: 앱 자체 업로드 큐를 통한 수동 동기화. Firebase Storage는 오프라인 큐잉 미지원)

### 지도 (Map)

- **FR-MAP-01:** 네이버 지도 전체 화면 (현재 위치, 줌)
- **FR-MAP-02:** 상태별 색상 마커 + InfoWindow (노랑/파랑/초록/주황/회색)
- **FR-MAP-03:** 상태별 필터 칩
- **FR-MAP-04:** 등록 단지 검색 + 위치 이동

### 공유/공동편집 (Share)

- **FR-SHARE-01:** 공유 링크 생성 (Editor/Viewer 권한 선택, 7일 만료)
- **FR-SHARE-02:** 공유 초대 수락 (딥링크, 미로그인 시 리다이렉트)
- **FR-SHARE-03:** Owner/Editor/Viewer 권한 관리
- **FR-SHARE-04:** 실시간 동기화 (5초 이내, LWW + 필드 병합)
- **FR-SHARE-05:** 활동 로그 (최근 50건)

### 공공데이터 연동 (PublicAPI)

- **FR-API-01~04:** 단지 목록/기본 정보/실거래가/건축물대장 API 연동 + Firestore 캐싱
- **FR-API-05:** 법정동코드 로컬 캐시 (앱 assets 번들 + SQLite, 약 5,000건)

## 인수 조건 (Acceptance Criteria)

### 인증

- 유효한 이메일/비밀번호로 가입/로그인 성공 시 홈 화면 이동. 비밀번호 확인 불일치 시 에러 메시지 표시
- Google/Apple 소셜 로그인 완료 시 홈 화면 이동, 최초 시 사용자 문서 자동 생성
- 로그인 상태 앱 재시작 후 유지

### 단지 관리

- 단지명 2글자 이상 입력 시 500ms 디바운스 후 자동완성 목록 표시 (단지명, 주소, 세대수)
- 등록 시 공공API 상세정보 자동 연동 + 중복 등록 방지
- 최근 6개월 실거래가 표시 (계약일, 면적, 가격, 층), API 실패 시 캐시 폴백

### 임장 기록

- 체크항목 5개 평점 + 텍스트 + 사진(최대 20장, 자동 압축) 저장 가능
- 오프라인에서 기록 작성 후 네트워크 복구 시 자동 동기화

### 지도

- 등록된 모든 단지가 상태별 색상 마커로 표시, 마커 탭 시 InfoWindow -> 상세 화면 이동
- 위치 권한 거부 시 기본 위치(서울 시청) 표시 + 권한 안내 메시지

### 공유

- 공유 링크로 초대된 사용자가 권한에 따라 읽기/쓰기 가능. 공유 링크는 7일 후 만료
- 한 사용자의 변경이 5초 이내에 다른 참여자에게 반영 (Wi-Fi 기준)

## 설계 방향 (Design Direction)

### 아키텍처

3계층 Clean Architecture 변형 (Presentation -> Domain -> Data), Feature-first + 내부 Layer 분리 폴더 구조.

```
lib/
├── app/          # MaterialApp, Router, Theme
├── core/         # 상수, 에러, 네트워크, 유틸
├── shared/       # 공통 위젯, 확장
├── features/     # auth, complex, inspection, map, share, public_api, region, settings
└── database/     # drift DB 정의
```

### 데이터 모델 (Firestore)

| 컬렉션 | 경로 | 설명 |
|--------|------|------|
| users | `users/{userId}` | 사용자 프로필 |
| complexes | `complexes/{complexId}` | 단지 정보 + `sharedWith` 배열 (쿼리 필터용) |
| inspections | `complexes/{cId}/inspections/{iId}` | 임장 기록 (서브컬렉션) |
| photos | `.../inspections/{iId}/photos/{pId}` | 사진 (서브컬렉션) |
| shares | `complexes/{cId}/shares/{sId}` | 공유 참여자/권한 |
| activityLogs | `complexes/{cId}/activityLogs/{lId}` | 활동 로그 (불변) |
| apiCache | `apiCache/{cacheKey}` | 공공API 응답 캐시 (TTL 기반) |

### 공유 모델

- **권한:** Owner (전체 권한) / Editor (기록 CRUD, 상태 변경) / Viewer (읽기 전용)
- **실시간 동기화:** Firestore 실시간 리스너, LWW + 필드 레벨 병합
- **딥링크:** App Links (Android) + Universal Links (iOS) + `app_links` 패키지

## QA 테스트 시나리오 (QA Test Scenarios)

### 시나리오 1: 단지 등록 ~ 임장 기록 작성 (핵심 플로우)

| 항목 | 내용 |
|------|------|
| **사전 조건** | 이메일 회원가입 완료, 로그인 상태 |
| **스텝** | 1. 홈 화면에서 (+) 버튼 탭 2. 지역 선택 (서울 > 강남구 > 대치동) 3. "래미안" 검색 4. 자동완성 목록에서 단지 선택 5. 단지 등록 완료 확인 6. 단지 상세 화면에서 기본 정보/실거래가 표시 확인 7. "임장 기록 작성" 탭 8. 방문일/시간대/체크항목 평점 입력 9. 사진 3장 촬영 첨부 10. 저장 |
| **기대 결과** | 단지 목록에 등록된 단지 표시, 임장 기록에 입력 내용 + 사진 정상 표시, 지도에 마커 표시 |

### 시나리오 2: 오프라인 임장 기록

| 항목 | 내용 |
|------|------|
| **사전 조건** | 단지 1개 이상 등록, 로그인 상태 |
| **스텝** | 1. 비행기 모드 활성화 2. 오프라인 배너 표시 확인 3. 기존 단지의 임장 기록 작성 (체크항목 + 사진 2장) 4. 저장 5. 기록 조회 확인 6. 비행기 모드 해제 7. 동기화 완료 토스트 확인 |
| **기대 결과** | 오프라인에서 기록 저장/조회 가능, 온라인 복귀 시 자동 동기화 완료, 다른 디바이스에서 확인 가능 |

### 시나리오 3: 공유 초대 + 실시간 동기화

| 항목 | 내용 |
|------|------|
| **사전 조건** | 사용자 A (Owner)가 단지 등록 완료, 사용자 B 계정 존재 |
| **스텝** | 1. A: 단지 상세 > 공유 > Editor 권한 > 링크 생성 2. B: 공유 링크 클릭 > 로그인 > 초대 수락 3. B: 단지 목록에 공유 단지 표시 확인 4. B: 임장 기록 작성 + 저장 5. A: 5초 이내에 B의 기록이 표시되는지 확인 6. A: B의 권한을 Viewer로 변경 7. B: 임장 기록 작성 버튼이 비활성화되는지 확인 |
| **기대 결과** | 공유 링크로 참여, 실시간 동기화, 권한 변경 즉시 반영 |

### 시나리오 4: 지도 탐색 + 필터

| 항목 | 내용 |
|------|------|
