# Feedback Loop

## 목적

사람이 AI 결과물에 피드백하면 AI가 재작업하는 흐름을 정의한다.

---

## 피드백 채널

- 요구사항 피드백: Jira 코멘트
- 코드 피드백: PR 코멘트

피드백 채널을 통일하여 AI가 일관되게 수집할 수 있도록 한다.

---

## 요구사항 피드백

```
AI 확장 결과 → 사람 검토 → 코멘트 (예: "범위가 너무 넓다")
→ AI가 스코프 조정 후 재확장 → 사람 재확인
```

---

## 계획 피드백

```
AI 작업 분해 → 사람 검토 → 코멘트 (예: "이 작업은 불필요하다")
→ AI가 계획 수정 → 사람 재확인
```

---

## 코드 피드백

```
AI PR 생성 → 사람 리뷰 → 코멘트 (예: "기존 패턴을 따라")
→ AI가 코드 수정 → 사람 재리뷰
```

---

## 재작업 규칙

- AI는 피드백을 받으면 자동으로 재작업 시작
- 최대 재작업 횟수: 3회
- 3회 초과 시 에스컬레이션 (09_escalation_policy 참조)
